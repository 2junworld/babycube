# 베이비큐브 — 배포 가이드

지금은 **1부(Vercel 테스트 배포)**만 진행하시면 됩니다. 앱이 실제 웹 주소로 잘 뜨는지, 폰에서 잘 쓰이는지 먼저 확인한 뒤, 나중에 부부 실시간 공유가 필요해지면 **2부(Firebase)**를 이어서 진행하시면 됩니다. 2부는 1부에서 만든 프로젝트에 이어 붙이는 구조라 지금 순서 그대로 진행하시면 됩니다.

> 이 폴더(`babycube`)에는 이미 프로젝트 파일(package.json, App.jsx 등)이 준비되어 있습니다. 아래 1단계의 `npm create vite` 과정은 이미 완료된 상태이니, **2단계(패키지 설치)부터** 시작하시면 됩니다.

# 1부 — Vercel 테스트 배포 (지금)

## 미리 알아둘 것 한 가지 (중요)

이 앱 코드는 레이아웃에 Tailwind CSS 클래스(`flex`, `items-center` 등)를 141곳에서 쓰고 있습니다. Claude 미리보기 화면에는 Tailwind가 미리 깔려 있어서 신경 쓸 필요가 없었지만, 일반 Vite 프로젝트에는 기본으로 없습니다. 아래 3단계에서 Tailwind를 설치하는 걸 건너뛰면 **버튼과 카드들이 가로 정렬 없이 세로로 다 풀려서 쌓이는** 형태로 깨져 보입니다. 순서대로 따라 하시면 문제없습니다.

## 1단계 — 프로젝트 만들기 (완료됨)

이 폴더가 이미 해당 프로젝트입니다. 터미널에서 이 폴더로 이동합니다:

```bash
cd babycube
```

## 2단계 — 패키지 설치

```bash
npm install
```

`package.json`에 react, react-dom, lucide-react, recharts, vite, tailwindcss, @tailwindcss/vite가 이미 등록되어 있어 이 한 줄로 전부 설치됩니다.

`vite.config.js`, `src/index.css`도 이미 Tailwind 설정이 반영된 상태로 준비되어 있습니다.

## 3단계 — 앱 코드 (완료됨)

`src/App.jsx`에 이유식 공유 앱 전체 코드가 이미 들어가 있습니다. 별도 작업이 필요 없습니다.

## 4단계 — 로컬에서 확인

```bash
npm run dev
```

터미널에 뜨는 `http://localhost:5173` 주소를 브라우저로 열어서, 폰 목업 화면이 원래 보던 모습 그대로(레이아웃 안 깨지고) 뜨는지 확인합니다. 여기까지 되면 배포 준비 끝입니다.

## 5단계 — Vercel로 배포

가장 간단한 방법(CLI):

```bash
npm install -g vercel
npm run build
vercel
```

안내에 따라 로그인하고 몇 가지 질문(프로젝트 이름 등)에 엔터만 쳐도 기본값으로 진행됩니다. 끝나면 `https://babycube-xxxx.vercel.app` 같은 주소가 나옵니다.

> GitHub을 이미 쓰신다면: 이 프로젝트 폴더를 GitHub 저장소로 올리고 [vercel.com](https://vercel.com)에서 "Import Project"로 그 저장소를 연결하는 방법도 있습니다. 이렇게 하면 이후 코드를 수정해서 push할 때마다 자동으로 재배포됩니다.

## 6단계 — 폰에서 앱처럼 쓰기

- **아이폰(Safari):** 주소 열기 → 공유 버튼 → "홈 화면에 추가"
- **안드로이드(Chrome):** 주소 열기 → 메뉴(⋮) → "홈 화면에 추가"

홈 화면 아이콘으로 추가하면 일반 앱처럼 아이콘 탭 한 번으로 열립니다. 지금 단계에서는 데이터가 각 기기의 브라우저(localStorage)에만 저장되므로, 아내분 폰에서 열면 별도의 빈 상태로 시작됩니다 — 이 부분이 2부(Firebase)에서 해결됩니다.

# 2부 — Firebase 계정 로그인 + 가족 공유 (구현 완료, 설정만 하면 됨)

Google 로그인 → 가족 만들기/6자리 초대 코드로 합류 → 데이터가 클라우드(Firestore)에 저장되는 구조가 이미 코드에 반영되어 있습니다. `src/firebase.js`, `src/App.jsx`(AuthGate, LoginScreen, FamilySetupScreen, FamilyStoreProvider, MembersScreen)에 전부 구현되어 있고, `package.json`에도 `firebase` 패키지가 등록되어 있습니다. 아래 순서대로 **Firebase 콘솔 설정 + 실제 키 값 입력**만 하시면 됩니다.

## 전체 그림

```
[ 이준세님 폰/PC ] ── Google 로그인 ──┐
                                      ├─→  Firestore families/{초대코드}  ←→ 실시간 동기화
[ 배우자 폰/PC ]   ── Google 로그인 ──┘        (state 필드에 모든 데이터 저장)
```

로그인 계정에 데이터가 묶여 있으므로, 브라우저 데이터를 지우거나 새 기기에서 로그인해도 그대로 이어집니다. 배우자분은 이준세님이 만든 6자리 초대 코드를 앱에 입력하면 같은 가족 데이터에 합류합니다.

## 1단계 — 패키지 설치

```bash
cd "C:\Users\wowju\Claude\Projects\육아\babycube"
npm install
```

(`package.json`에 firebase가 이미 등록되어 있어 이 한 줄로 설치됩니다.)

## 2단계 — Firebase 프로젝트 만들기

1. https://console.firebase.google.com 접속 → "프로젝트 추가"
2. 프로젝트 이름(예: `babycube`) 입력, 애널리틱스는 꺼도 됨
3. 생성 후 왼쪽 메뉴 **빌드 → Firestore Database → 데이터베이스 만들기**
   - 위치는 `asia-northeast3 (서울)` 권장
   - 모드는 **테스트 모드**로 시작해도 되지만, 이 프로젝트는 4단계의 보안 규칙을 바로 적용하시길 권장합니다.
4. 왼쪽 메뉴 **빌드 → Authentication → 시작하기 → Sign-in method 탭 → Google** 사용 설정 (프로젝트 지원 이메일은 본인 Gmail 선택)
5. 왼쪽 위 톱니바퀴 → **프로젝트 설정** → 하단 "내 앱"에서 **웹 앱(</>)** 추가 (앱 닉네임은 아무거나, Firebase Hosting 체크는 안 해도 됨)
6. 표시되는 `firebaseConfig` 값들을 복사해 둡니다:

```js
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "babycube-xxxx.firebaseapp.com",
  projectId: "babycube-xxxx",
  storageBucket: "babycube-xxxx.appspot.com",
  messagingSenderId: "...",
  appId: "..."
};
```

## 3단계 — 앱에 키 값 붙여넣기

`src/firebase.js` 파일을 열어서, 위에서 복사한 6개 값을 `여기에_...` 자리에 그대로 교체합니다. 코드 구조는 이미 완성되어 있어 이 값만 바꾸면 됩니다.

## 4단계 — 보안 규칙 적용 (중요)

Firestore → **규칙** 탭에 이 프로젝트에 이미 포함된 `firestore.rules` 파일 내용을 붙여넣고 **게시**를 누릅니다.

이 규칙은: 로그인한 사용자만 가족 문서를 읽을 수 있고, 가족 생성은 본인 1명만 멤버로 넣을 때만, 초대 코드로 합류하는 것은 본인 1명만 추가되는 경우에만 허용합니다. 로그인이 필요하므로 이전(테스트 모드) 방식보다 안전하지만, 초대 코드를 아는 사람은 누구나 합류할 수 있는 구조이니 코드는 배우자분에게만 공유해 주세요.

## 5단계 — 로컬에서 확인

```bash
npm run dev
```

`http://localhost:5173` 접속 → "Google로 로그인" → "새 가족 만들기"를 눌러 6자리 초대 코드가 뜨는지 확인합니다. 그 코드를 메모해 두었다가, 다른 브라우저(또는 시크릿 창)에서 다른 Google 계정으로 로그인 → "초대 코드로 합류하기"에 입력해서 같은 데이터가 뜨는지, 한쪽에서 수정하면 다른 쪽에도 반영되는지 확인해 보세요.

## 6단계 — 재배포

```bash
npm run build
vercel --prod
```

(GitHub 연동 방식이면 push만 하면 자동 재배포됩니다.)

## 배우자분 사용 방법

1. 배포된 주소로 접속 → "Google로 로그인" (본인 Gmail 계정)
2. "초대 코드로 합류하기" → 이준세님이 알려준 6자리 코드 입력
3. 이후부터는 로그인만 하면 항상 같은 가족 데이터가 뜨고, 실시간으로 공유됩니다.
